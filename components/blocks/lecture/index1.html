初识提示词工程

chat 什么是提示词？

提示词（Prompt）就是你向人工智能发出的指令、问题或者任何形式的输入。它是你启动与AI对话、引导AI进行思考和生成内容的"钥匙"或"启动器"。

类比理解

你可以把AI想象成一个知识渊博，但需要清晰指引才能行动的超级助手。你的提示词，就好比是给这位助手下达的"工作简报"或"任务说明书"。

形式多样

提示词可以是一个简单的词、一个直接的问题、一段详细的描述，甚至包含背景信息、具体要求和角色扮演的指令。

engineering 什么是提示词工程？

提示词工程（Prompt Engineering）是一门关于如何更好地设计和优化你给AI的指令（提示词）的学问，目的是为了让AI能够给出最有用、最准确、最符合你预期的回答或输出。

核心理念

一个核心的原则是"输入决定输出"（Quality In, Quality Out）。你提供给AI的提示词的质量越高、越清晰、越有针对性，AI反馈给你的结果通常就会越好。

tips_and_updates

提示词工程往往是一个反复试验和迭代优化的过程。不要害怕调整和改进你的提示词！

掌握核心概念

clear_all

使用明确的行动动词，避免含糊词语，聚焦核心任务，确保AI准确理解你的意图。

info

提供上下文

说明目标受众，提供关键背景信息，引用特定来源，让AI能够生成高度相关的内容。

角色扮演

给AI指定一个特定的角色或身份，使其回答更贴合该角色的知识水平和语言风格。

指定格式

明确要求特定的输出结构、长度限制、语气和风格，减少后期整理工作。

view_module PST-R：提示词设计原则

为了帮助我们更有条理地设计出高效的提示词，可以借鉴PST-R这个思考框架。它包含四个核心要素：

P - 目的 (Purpose)

你到底想让AI做什么？是总结信息、创作故事、分析数据还是其他任务？明确目标是第一步。

S - 空间 (Space)

AI的"思考空间"如何？需要提供哪些上下文信息来限定它的思考范围？如何定义输出格式？

T - 时间 (Time)

任务需要一次性完成还是多轮对话？AI是否需要按特定时间顺序思考或组织内容？

R - 角色 (Role)

希望AI以什么样的身份或视角来回应？设定合适的角色能显著提升输出内容质量。

PST-R框架不仅是一个设计清单，它实际上揭示了提示词工程各种技巧之所以有效的内在结构。在编写提示词时对照这四个维度进行检查，可以系统地确保所有关键要素都得到考虑。

解锁AI潜能：常用技巧

stairs 逐步思考：引导AI深度推理

当面对复杂问题时，引导AI"一步一步地思考"能显著提高答案的准确性和逻辑性。这种被称为"思维链"（Chain of Thought, CoT）的技巧，会引导AI模拟人类解决问题时的推理步骤。

示例：弹珠计算问题

person
假设我最初有8个弹珠。我给了朋友3个，然后我又找到了4个。我现在一共有多少个弹珠？请一步一步思考。
smart_toy
  1. 初始状态：我有8个弹珠。
  2. 第一个动作：给了朋友3个。
  3. 计算剩余数量：8 - 3 = 5个弹珠。
  4. 第二个动作：又找到了4个。
  5. 计算最终数量：5 + 4 = 9个弹珠。
  6. 结论：所以，我现在一共有9个弹珠。

适用场景：数学应用题、逻辑谜题、复杂的因果分析、多步骤项目规划等。

content_copy 提供示例：让AI模仿学习

与其用很多文字描述你想要什么，不如直接给AI看几个例子。这种方法被称为"少样本提示"（Few-Shot Prompting），利用了大型语言模型从少量示例中学习和泛化的能力。

示例：情感分类

将以下评论分类为"积极"、"消极"或"中性"。

评论： "这部电影简直是浪费时间。" 情感：消极
评论： "我全程笑得停不下来！" 情感：积极
评论： "特效很棒，但故事情节有点混乱。" 情感：中性
评论： "我觉得这家餐厅的服务还可以。" 情感：？

编写技巧

格式一致性： 确保所有示例遵循相同的输入/输出格式。

示例多样性： 选择能覆盖不同情况的示例，帮助AI更好地泛化。

示例顺序： 模型对提示词中最后出现的示例可能更敏感。

分隔符： 使用清晰的分隔符将指令、示例和输入区分开。

dashboard 生成多模态学习对象

提示词不仅能用来生成文本，还可以引导AI创造各种不同形式的学习资源，满足不同学习偏好和教学场景的需求。

summarize

生成文本摘要

用提示词让AI快速提炼冗长文章的核心内容。

"请将这篇关于'全球变暖影响'的5000字研究报告，总结成一个300字左右的关键点摘要，面向政策制定者。"
code

生成代码示例

用提示词让AI生成特定功能的代码片段，辅助学习编程。

"请用Python编写一个函数，输入一个整数列表，返回列表中所有偶数的平均值，并添加适当的注释说明代码逻辑。"
quiz

生成测验题目

教师可以利用AI快速生成用于评估学生理解程度的测验题。

"根据以上提供的关于'细胞结构'的学习材料，请生成5道单项选择题，每题包含四个选项，并明确标出正确答案和简要解释。"

提示词工程是实现学习内容多样化和个性化的强大杠杆。它极大地降低了创建多样化教学内容的门槛，使得原本可能需要大量时间和专业技能才能完成的工作变得触手可及。

超越基础：创意与思考

psychology_alt 激发知识情感

学习远不止是冷冰冰的知识获取，情感在其中扮演着至关重要的角色。某些特定的情感状态与学习、探索和反思行为紧密相关，它们被称为"知识情感"（Knowledge Emotions）。

激发惊喜 (Surprise)

惊喜源于预期之外的事物，可以要求AI提供与普遍认知相悖的"反直觉"事实。

"请告诉我5个关于古埃及金字塔建造的、最令人惊讶且有据可查的事实，这些事实可能会颠覆普通人的普遍认知。"

激发兴趣 (Interest)

兴趣是持续学习的强大动力，可以让AI生成与学习者自身兴趣点联系的内容。

"假设你是一位热爱篮球的体育记者，请用生动有趣的方式，向同样喜欢篮球的学生解释物理学中的'抛物线运动'原理。"

激发好奇 (Curiosity)

好奇心驱使我们探索未知，可以要求AI生成有启发性的开放性问题或谜题。

"关于'人类意识的本质'这个哲学问题，目前科学和哲学领域存在哪些主要的未解之谜？请生成3个能够引导学生进行深入探究和讨论的问题。"

激发敬畏 (Awe)

敬畏感通常来自于面对宏大、壮丽或极其复杂深刻的事物，能让我们感到自身的渺小。

"请扮演一位诗人兼天文学家，用充满敬畏和想象力的语言，描绘从詹姆斯·韦伯太空望远镜传回的创生之柱图像所展现的宇宙景象。"

tune 带宽-即时性矩阵与提示词

在线教育领域常用的"带宽-即时性矩阵"工具可以类比到提示词工程中，帮助我们根据不同的情境和需求来设计和使用提示词。

提示词复杂度/长度 (类比带宽)

低复杂度/短提示：

简单、直接的提示词通常需要AI动用的"计算资源"较少，响应速度更快，适合需要快速获取信息的场景。

高复杂度/长提示：

包含大量背景信息、多步骤指令的复杂提示词需要AI投入更多"计算资源"，适合生成详细的内容或解决复杂问题。

交互需求 (类比即时性)

高即时性需求：

在需要快速得到AI反馈的场景下，如实时课堂问答，更适合使用简洁直接的提示词。

低即时性需求：

在允许AI有更多时间处理的场景下，如异步学习任务，可以使用更复杂、更详尽的提示词。

核心思想：提示词工程并非只有一种"正确"的方法，有效的提示策略需要根据具体的交互场景、预期响应以及可用资源进行灵活调整。

动手实践：轮到你来提示！

理论学习是基础，但真正掌握提示词工程的关键在于实践。以下练习将帮助你将前面学到的概念和技巧付诸行动。

1
提升清晰度与上下文

针对一个你熟悉的主题，通过改进提示词的清晰度和上下文信息，来提升AI生成内容的质量。

  1. 选择一个你比较了解的话题
  2. 编写一个模糊的初始提示词，记录AI输出
  3. 运用"清晰指令"和"提供上下文"原则改进提示词
  4. 再次生成内容并对比结果
  5. 反思两次输出的差异

2
体验角色扮演

让AI扮演不同的角色来解释同一个概念，观察输出风格和内容的变化。

  1. 挑选一个稍微复杂的概念（如"光合作用"）
  2. 让AI不带任何角色地解释这个概念
  3. 让AI扮演"小学科学老师"解释给8岁孩子听
  4. 让AI扮演"领域专家"在学术会议上解释
  5. 对比三次输出的词汇、风格、深度和例子选择上的差异

3
运用示例学习 (Few-Shot)

通过提供少量示例，教会AI模仿一种特定的写作风格或遵循一种特定的输出格式。

  1. 选择一种你希望AI模仿的风格或格式
  2. 准备2-3个高质量的输入/输出示例
  3. 构建Few-Shot提示词，包含任务指令和示例
  4. 评估AI生成的内容是否成功模仿了目标风格
  5. 如需要，调整示例或指令后再次测试
